# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within Doser, please send an email to security@doser.app. All security vulnerabilities will be promptly addressed.

Please include the following information in your report:
- Type of vulnerability
- Full paths of source file(s) related to the vulnerability
- Location of affected source code (tag/branch/commit or direct URL)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## Security Measures

Doser implements the following security measures:

1. **Data Protection**: All account data is stored locally
2. **No External Transmission**: No data is sent to external servers
3. **Proxy Support**: Built-in proxy rotation for anonymity
4. **Anti-Detection**: Uses undetected-chromedriver to avoid detection

## Best Practices

When using Doser:

1. Use strong, unique passwords
2. Enable 2FA on your Telegram account
3. Use reputable proxy services
4. Keep your system and browser updated
5. Don't create too many accounts in a short time
